1.	Что такое секционирование таблиц?
Секционирование (partitioning) — это метод разбиения большой таблицы (или индекса) на более мелкие, логически независимые части — секции (partitions).
 Для клиента и SQL‑запросов таблица выглядит как единое целое, но СУБД хранит и управляет каждой секцией отдельно.
  Это повышает производительность управления данными, упрощает администрирование и даёт возможности для масштабирования и параллельной обработки.
  
2.	В каких случаях целесообразно применять секционирование?
Очень большие таблицы с миллионами/миллиардами строк (OLTP/OLAP).
Данные, естественно разделяющиеся по диапазонам времени (например, дневные/месячные партиции).

3.	Объясните принцип секционирования для всех типов секционирования, которые использовались в заданиях лабораторной работы.
Range (диапазонное)
    Принцип: строки помещаются в партицию на основании принадлежности значения ключа к определённому диапазону (например, даты).
    
List (списковое)
    Принцип: для каждой партиции явно перечисляются допустимые значения ключа (например, список регионов или кодов).
    
Hash (геширование)
    Принцип: значение ключа хешируется, и строки распределяются по N партициям равномерно. Нет семантической связи между партициями.
    
Interval (интервальное, Oracle)
    Принцип: автоматическое создание диапазонных партиций фиксированного интервала (например, по месяцам) при вставке данных вне существующих границ.
   
4.	Перечислите названия типов секционирования, которые не использовались в заданиях лабораторной работы. 
Range (в коде — T_RANGE)
Принцип: каждая строка попадает в партицию в зависимости от значения ключа (здесь — id) и границ диапазонов. Например, id < 100 → p0; 100 ≤ id < 200 → p1; остальные → PMAX.

Interval (в коде — T_INTERVAL)
Принцип: это разновидность range‑партиционирования, где партиции автоматически создаются при вставке значений за пределами существующих границ с фиксированным интервалом (здесь — 1 месяц).
В коде есть начальная партиция p1 (< 2024-01-01). При вставке дат после текущей верхней границы СУБД автоматически создаст новые месячные партиции.

Hash (в коде — T_HASH)
Принцип: хеш-функция по значению ключа (code) распределяет строки примерно равномерно по заданному числу партиций (4). 
Нет смысловой границы — цель равномерная балансировка вставок и уменьшение конфликтов блокировок.

List (в коде — T_LIST)
Принцип: для каждой партиции явно перечислены значения ключа (status). Например, status = 'A' → p_a; 'B' → p_b; остальные → p_other (DEFAULT).

5.	Объясните действие оператора ALTER TABLE MERGE.
Назначение: объединяет одну или несколько партиций в новую (или в соседнюю) партицию.
При слиянии объединяется содержимое перечисленных партиций; новая партиция получает общий диапазон значений.

6.	Объясните действие оператора ALTER TABLE SPLIT.
Назначение: разделяет одну партицию на две (или более) в соответствии с новыми границами.
Создаёт новые партиции с заданными диапазонами; данные, попадающие в соответствующие диапазоны, перераспределяются в новые партиции.

7.	Объясните действие оператора ALTER TABLE EXCHANGE.
Назначение: обмен содержимым партиции и отдельной (непартицированной) таблицы. Часто используется для быстрой загрузки/выгрузки данных.
Обычно операция быстрейшая, т.к. часто выполняется на уровне метаданных (без массового перемещения строк).
