1.	Что такое словарь данных Oracle? Для чего он необходим?

Словарь данных Oracle — это набор таблиц и представлений, содержащих метаданные, описывающие структуру базы данных, объекты, пользователей, привилегии и конфигурацию.
Он необходим для:
  -Хранения информации о всех объектах и их свойствах в базе данных.
  -Обеспечения управления доступом и безопасности данных.
  -Служит основным инструментом для выполнения запросов о структуре и состоянии базы данных.
  
  
2.	Проклассифицируйте представления словаря данных Oracle.

Представления словаря данных в Oracle классифицируются на три типа:
  -All Views: Дает доступ к метаданным для всех объектов, к которым у пользователя есть доступ.
  -User Views: Показывает метаданные только для объектов, созданных пользователем.
  -Dba Views: Дает полный доступ к метаданным для всех объектов в базе данных (требует соответствующих привилегий).
  
  
3.	Что значит Oracle 12c имеет клиент-серверную архитектуру?
Клиент-серверная архитектура в Oracle 12c подразумевает, что база данных (сервер) и клиентские приложения взаимодействуют через сетевое соединение. 

4.	Что такое экземпляр (инстанс) сервера СУБД?
Экземпляр сервера (инстанс) — это набор процессов, работающий в памяти и связанных с базой данных, отвечающий за управление данными и выполнение операций. 

5.	Перечислите все этапы старта и останова инстанса Oracle 12c. Поясните их назначение.

Этапы запуска инстанса:
    Initialize: Чтение файла инициализации (PFILE или SPFILE) для загрузки параметров конфигурации.
    Mount: Присоединение к базе данных, проверка структуры.
    Open: Открытие базы данных для доступа пользователей.

Этапы остановки инстанса:
    Shutdown: Остановка новых подключений к базе данных.
    Close: Закрытие базы данных.
    Terminate: Остановка всех процессов и освобождение ресурсов.

7.	Поясните назначение опции Oracle 12c Multitenant.

Oracle 12c Multitenant позволяет создавать и управлять несколькими подключаемыми базами данных (PDB) в рамках одной контейнерной базы данных (CDB)

8.	Поясните назначение баз данных CDB, PDB, PDB$SEED.

    CDB (Container Database): Это корневая база данных, которая содержит все метаданные и управляет всеми PDB.
    PDB (Pluggable Database): Подключаемая база данных, содержащая собственные данные и объекты, изолированные от других PDB.
    PDB$SEED: Шаблон для создания новых PDB, содержащий базовые объекты и настройки, необходимые для новой плавающей базы данных.
    

9.	Что такое общий пользователь Oracle 12c Multitenant?

Общий пользователь — это пользователь, который имеет доступ ко всем базам данных и объектам в CDB. 
Эти пользователи создаются в корневой базе данных и могут выполнять действия в PDB по мере необходимости. Примеры: SYS, SYSTEM.

10.	Какие табличные пространства автоматически создаются при инсталляции СУБД Oracle 12c?

При установке Oracle 12c создаются следующие табличные пространства:

    SYSTEM: Хранит данные для словаря данных.
    SYSAUX: Дополнительное пространство для поддержки функциональности базы данных.
    TEMP: Временное табличное пространство для операций, требующих временного хранения данных.

11.	Какие 2 пользователя обязательно создаются при инсталляции СУБД Oracle 12c?

При установке Oracle 12c обязательно создаются:

    SYS: Административный пользователь с полными привилегиями.
    SYSTEM: Второй административный пользователь, используемый для управления базой данных и выполнения повседневных задач.
    
    
12.	Какие табличные пространства дублируются в каждой PDB?

Каждая PDB получает свои собственные копии следующих табличных пространств:

    SYSTEM
    SYSAUX

13.	Что такое консолидация баз данных? В каких случаях прибегают к консолидации баз данных?
Консолидация баз данных — это процесс объединения нескольких баз данных в одну среду, что позволяет более эффективно использовать ресурсы, 
упростить администрирование и снизить затраты на инфраструктуру. Обычно консолидацию применяют в крупных организациях с множеством разрозненных баз данных для оптимизации управления.


14.	Какие преимущества дает Oracle 12c Multitenant?

Преимущества Oracle 12c Multitenant включают:

    Упрощенное администрирование: Управление несколькими PDB из единого CDB.
    Эффективное использование ресурсов: Совместное использование ресурсов между PDB.
    Быстрое развертывание: Легкое создание и клонирование баз данных.
    Безопасность и изоляция: Каждая PDB изолирована, что повышает безопасность данных.
