3.	Что такое табличное пространство?

Табличное пространство – логическая структура хранения данных, контейнер сегментов

4.	Перечислите типы табличных пространств и их назначение.

PERMANENT - Постоянное пространство. Таких может быть нексколько , можно создавать свои объекты в любом из PERMANENT пространств, если это не запрещено.
Одно постоянное пространство может быть приписано в качестве пространства по умолчанию

TEMPORARY - Временное простаранство. Таких может быть несколько , временное пространство может приписываться одному или нескольким пользователям и они могут 
хранить там свои временные данные

UNDO - Таких пространств может быть несколько, но активным является всегда только одно (указывается в конфигурационном файле SPIFLE.ORA)


5.	В каком соотношении находятся табличные пространства и файлы данных?
С одним табличным пространством связаны один или несколько файлов операционной системы,
с каждым файлом связано только одно табличное пространство

8.	Что такое роль и для чего она применяется?

Роль – это именованный набор привилегий
Привилегия - это право выполнять конкретный тип предложений SQL, или право доступа к объекту другого пользователя.  
ORACLE  имеет два   вида   привилегий:   системные  и  объектные.
Назначаются оператором GRANT
Отзываются оператором REVOKE


9.	Какие системные роли вы знаете? Объясните их назначение.

  8.1 SYSDBA - Полный доступ к базе данных; управление пользователями, привилегиями и общими ресурсами.
  8.2 SYSOPER - Тоже самое что sysdba, но только мониторинг
    
10.	Что такое системная привилегия, какие системные привилегии вы знаете
 
 Системная привилегия в Oracle — это право, предоставляемое пользователям или ролям для выполнения определённых действий в базе данных. 
 Системные привилегии позволяют выполнять операции на уровне всей базы данных, а не только для определённых объектов.
 
   9.1 CREATE SESSION
   9.2 CREATE TABLE/VIEW/PROCEDURE
   9.3 DROP ANY TABLE/VIEW/PROCEDURE
   9.4 GRANT ANY /REVOKE ANY


11.	Какие специальные системные привилегии администратора БД вы знаете? Как их назначить?(СКОРЕЕ ВСЕГО НЕПРАВИЛЬНО, ВРОДЕ НЕТ ТАКИХ)

Специальные системные привилегии администратора базы данных (БД) в Oracle позволяют выполнять критически важные операции с данными и структурой базы.

    ALTER DATABASE
        Позволяет изменять параметры БД, такие как режимы работы или добавление данных.

    FLASHBACK DATABASE
        Позволяет восстанавливать БД до предыдущего состояния.

    CREATE ANY DIRECTORY
        Позволяет создавать директории для работы с файлами вне базы данных.

    CREATE ANY JOB
        Позволяет создавать задания в Oracle Scheduler.

    ALTER ANY SCHEMA
        Позволяет изменять схемы, включая управление объектами внутри них.

    MANAGE SANDBOX
        Позволяет управлять песочницами для тестирования изменений без влияния на основную БД.
        
        

12.	Что такое профиль безопасности и для чего он нужен?
Профиль безопасности в Oracle — это набор параметров, определяющих ограничения и параметры учетных записей пользователей.
 Он предназначен для управления доступом к базе данных и повышения ее безопасности.


13.	Перечислите известные вам параметры профиля безопасности и поясните их.
 12.1 PASSWORD_LIFE_TIME - Время жизни пароля в днях
 12.2 SESSIONS_PER_USER - Количество сессий для пользователя
 12.3 FAILED_LOGIN_ATTEMPTS - Количество попыток входа
 12.4 PASSWORD_LOCK_TIME - Количество дней блокировки пароля после ошибок
 12.5 PASSWORD_REUSE_TIME - Через сколько дней можно повторить пароль
 12.6 PASSWORD_GRACE_TIME - Количество дней предупреждений о смене пароля
 12.7 CONNECT_TIME - Время соединения в минутах
 12.8 IDLE_TIME - Количество минут простоя


14.	Объясните назначение профиля с именем DEFAULT.
Профиль с именем DEFAULT в Oracle является предопределенным профилем, который автоматически назначается всем пользователям, 
не указывающим специального профиля при создании учетной записи.
Профиль DEFAULT обеспечивает минимальные, но необходимые уровни безопасности и управления ресурсами для всех пользователей базы данных. 
Это позволяет эффективно контролировать и защищать базу данных.

15.	Что такое пользователь базы данных?
Пользователь базы данных — это учетная запись, которая позволяет взаимодействовать с базой данных, выполняя кикие-либо операции.
То есть учётная запись, которой гарантирована какая-либо роль(либо дефолтная, либо созданная)
