1. Дайте пояснению понятию  «событие программного объекта». 
  
Событие в  программном  объекте – это  переход  объекта из одного состояние в другое. 
При этом,  об этом переходе могут быть извещены другие объекты. У события есть издатель (или генератор) события и могут быть подписчики (или обработчики) события.
  
2. Объясните механизм генерации и обработки событий в C#.
  
В C#, механизм генерации и обработки событий основан на использовании делегатов и специализированного ключевого слова event.
Этот механизм позволяет объектам взаимодействовать друг с другом, оповещая их о некоторых изменениях состояния.
   
Делегат — это тип, представляющий ссылки на методы с определенным набором параметров и возвращаемым значением.
Делегаты используются для определения сигнатуры метода, который будет вызван в ответ на событие.
 
 
Событие — это особый тип делегата, связанного с каким-либо действием, происходящим в объекте.
События объявляются с использованием ключевого слова event.

События генерируются в ответ на определенные действия или состояния. Это может быть вызов метода, изменение свойства или какое-то другое действие.

4. Какой встроенный механизм используется в Node.js для генерации и обработки событий. Поясните принцип его работы.   
EventEmitter: JS-класс, предоставляющий функциональность для асинхронной обработки событий в Node.js.
необходимо включения двух модулей: events и util.
Как правило, применяется в качестве базового для пользовательского объекта. Производный  от EventEmitter объект может быть создан с помощью  функции inherits модуля  utils.   
Производный от EventEmitter объект приобретает функциональность, позволяющую генерировать и прослушивать события. 
Для генерации событий предназначена функция emit, а для прослушивания функция on. 

От Смелова:
1. CORS
CORS - Cross Origin Resource Sharing -это функция безопасности, реализованная в веб-браузерах, которая позволяет или запрещает запросы на ресурсы из разных источников(origins) 
(т.е., с различными именами доменов, протоколами или портами).

Зачем нужно:
Безопасность: CORS помогает защитить данные пользователя от несанкционированных кросс-оригинальных запросов и потенциальных атак, таких как подделка межсайтовых запросов (CSRF).
Гибкость для разработчиков: Он позволяет разработчикам указывать, какие домены могут получать доступ к их ресурсам, тем самым обеспечивая легитимное кросс-оригинальное взаимодействие
(например, API) и предотвращая злонамеренные действия.
Пример сценария

Предположим, есть фронтенд-приложение, размещенное на http://localhost:3000, и API REST на http://localhost:5000.
Без CORS, если ваш фронтенд попытается получить данные с API, браузер заблокирует запрос. Но если API ответит Access-Control-Allow-Origin: http://localhost:3000, браузер разрешит запрос.

2. mode:no-cors
В API fetch параметр mode определяет режим запроса.Если устанавдивается 'no-cors', это значит, что тело и заголовки запроса недоступны для JavaScript
(то есть отключается cors-механизм).
Это подходит для простых запросов (по типу GET) для которых нам не нужно получать доступ к телу или заголовкам.
