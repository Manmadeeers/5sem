1. Поясните разницу между полудуплексном и дуплексным  каналами данными.
Полудуплекс (half‑duplex) — канал, в котором в любой момент времени данные могут передаваться 
только в одном направлении: либо A→B, либо B→A. Передача в обоих направлениях возможна, но поочерёдно (пример: рация).

Дуплекс (full‑duplex) — канал, позволяющий одновременно передавать данные в обоих направлениях (A↔B одновременно), 
без ожидания освобождения линии (пример: телефонный разговор).

2. Какой тип канала применяется HTTP‑протоколом

На сетевом уровне HTTP обычно работает поверх TCP, который физически предоставляет full‑duplex (полный дуплекс).
На уровне приложения традиционный HTTP (до HTTP/2) реализует модель запрос‑ответ, где клиент инициирует запросы, а сервер отвечает 
— семантически это похоже на полудуплексный обмен (инициатива у клиента). Но это не значит, что данные не могут идти одновременно по сети — TCP позволяет.

3. Пояснение понятия WebSocket

WebSocket — это протокол для установки постоянного двунаправленного соединения между браузером (клиентом) и сервером.
После установки WebSocket позволяет обмениваться фреймами произвольной длины в обе стороны без накладных расходов HTTP‑заголовков для каждого сообщения — 
подходит для realtime‑приложений (чаты, игры, уведомления).
WebSocket поддерживает текстовые и бинарные фреймы, пинг/понг для проверки живости, и обычно работает поверх TCP (либо TLS поверх TCP для wss://).

4. Какой тип канала применяется WebSocket‑протоколом

WebSocket предоставляет full‑duplex (двунаправленный одновременно) канал на уровне приложения: 
и клиент, и сервер могут в любой момент инициировать отправку сообщений по установленному соединению.

Это постоянное соединение (persistent connection), в отличие от классического HTTP request/response.


5. Пояснение «широковещательное сообщение» (broadcast) и «широковещательный сервер»

Широковещательное сообщение (broadcast) — сообщение, которое отправляется одному источнику с последующей доставкой сразу множеству получателей (группе или всем подключённым клиентам).

Широковещательный сервер — сервер, реализующий логику отправки таких сообщений: принимает входные события (или генерирует их) и распределяет (браodcast) их по множеству подключённых клиентов
