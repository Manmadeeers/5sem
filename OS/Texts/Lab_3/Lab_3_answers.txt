1. Что такое процесс?
Процесс – управляющий объект, который обеспечивает изоляцию адресных пространств и представляет работающий экземпляр программы
Процесс – исполняемое на устройстве приложение весте со всеми ресурсами, которые требуются для его исполнения

2. Какие ресурсы доступны процессу?
-Образ исполняемого файла
-Память
-Список дескрипторов объектов выделенных процессу (файлы, потоки, объекты синхронизации и т.д.)
-Атрибуты безопасности (маркеры)
-Контекст процесса

3. Что такое дочерний и родительский процесс?
Дочерний процесс – процесс создаваемый в результате системного вызова
Родительский процесс – процесс инициировавший системный вызов
  
4. Что такое системный вызов?
обращение прикладной программы к ядру операционной системы для выполнения какой-либо операции
(За создание процесса отвечает ядро ОС
Чтобы запросить у ядра создание нового процесса необходимо совершить системный вызов)

5. Какие системные вызовы предназначены для создания процессов в Win-dows? Linux?
 Для Linux: fork(), vfork()
 Для Windows: NtCreateUserProcess
 
 6. Какие функции из WinAPI и POSIX связаны с ранее названными систем-ными вызовами?
 Для Windows: CreateProcessA(), CreateProcessW()  /  WaitForSingleObject(), WaitForMultipleObjects  /  ExitProcess(), TerminateProcess(), CloseHandle()
 Для Linux: fork(), exec()  /  wait()  /  exit()

8. Для чего нужен exec?
предназначена для замены образа исполняемого файла в рамках существующего процесса. Никаких новых процессов при вызове exec не создаётся. Загружается новая программа!
